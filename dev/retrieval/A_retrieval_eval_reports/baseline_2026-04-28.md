## Sweep Findings

### Mode (7 modes)

| Mode | Doc Recall @10 | Snippet Recall @10 |
|------|---------------|-------------------|
| dense | 77% | 72% |
| sparse | 79% | 70% |
| hybrid (RRF) | 80% | 75% |
| **cc** | **80%** | **78%** |
| cc+rerank | 84% | 77% |
| hybrid+rerank | 84% | 77% |
| bm25 | 76% | 53% |

`cc` is the best non-reranking mode at 80% Doc / 78% Snippet Recall. Adding a reranker trades +4pp Doc Recall (80%→84%) for −1pp Snippet Recall (78%→77%); that trade-off is rejected for MCP tool responses where snippet quality dominates. `cc+rerank` and `hybrid+rerank` converge to identical results — the reranker normalizes fusion-input differences.

### Alpha (5 values: 0.5–0.9)

| α | Doc Recall @10 | Snippet Recall @10 |
|---|---------------|-------------------|
| 0.5 | 80% | 73% |
| 0.6 | 80% | 73% |
| 0.7 | 80% | 75% |
| **0.8** | **80%** | **78%** |
| 0.9 | 80% | 72% |

Doc Recall is constant at 80% across all alpha values. Snippet Recall peaks clearly at α=0.8 — confirming CC_ALPHA=0.8 as the optimal default. Values below or above 0.8 degrade Snippet Recall.

### top_k (3 values)

| top_k | Doc Recall | Snippet Recall |
|-------|-----------|---------------|
| 5 | 74% | 57% |
| 10 | 80% | 78% |
| **20** | **86%** | **83%** |

top_k has a strong monotonic effect: each doubling from 5→10→20 adds +6–8pp on both metrics. At top_k=20 the pipeline is measured at the same operating point as Production (which clamps `max(top_k, 20)`). The previous BASELINE.top_k=10 understated real production performance by 6pp Doc / 5pp Snippet. BASELINE.top_k is updated to 20.

### rrf_k (3 values)

| rrf_k | Doc Recall @10 | Snippet Recall @10 |
|-------|---------------|-------------------|
| 30 | 80% | 75% |
| 60 | 80% | 75% |
| 90 | 80% | 75% |

K-insensitive: all three values produce identical results on this collection. The RRF_K=60 default is retained as the standard value but has no practical effect within this range.

### score_threshold (3 values)

| threshold | Doc Recall @10 | Snippet Recall @10 |
|-----------|---------------|-------------------|
| 0.0 | 80% | 78% |
| 0.3 | 80% | 78% |
| 0.5 | 80% | 78% |

No effect on CC scores within the tested range [0.0, 0.3, 0.5]. CC fusion scores are not interpretable as probabilities; hard thresholds do not filter meaningfully. BASELINE.score_threshold=0.0 (no filtering) confirmed.

### query_prefix (True / False)

| query_prefix | Doc Recall @10 | Snippet Recall @10 |
|-------------|---------------|-------------------|
| **True** | **80%** | **78%** |
| False | 79% | 72% |

Enabling the Instruct query prefix (`Instruct: Given a search query, retrieve relevant passages that answer the query\nQuery: `) adds +6pp Snippet Recall. The Qwen3-Embedding-8B model benefits from the prefix for asymmetric retrieval (short query vs longer chunks). query_prefix=True is confirmed as the correct default.
